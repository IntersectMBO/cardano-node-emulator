# Support

Thanks for using the Cardano Node Emulator! This document explains where to get
help and how to report problems.

> **Note:** This is pre-release software. We do not usually provide dedicated
> support, and interfaces may change between versions. Community help is offered
> on a best-effort basis.

## Documentation

Before asking for help, please check the available documentation:

- The [README](README.md) for an overview of the project and its components.
- [CONTRIBUTING.md](CONTRIBUTING.md) for development setup and contribution
  guidelines.
- The Haskell API documentation (haddocks), which you can generate from within
  `nix develop`, for example:

  ```
  cabal haddock cardano-node-emulator
  ```

## Asking Questions

For usage questions, ideas, and general discussion, please open a
[GitHub issue](https://github.com/IntersectMBO/cardano-node-emulator/issues).
Search the existing issues first in case your question has already been answered.

## Reporting Bugs

If you have found a bug, please file it in the
[GitHub issue tracker](https://github.com/IntersectMBO/cardano-node-emulator/issues).
A good bug report includes:

- The affected version or commit.
- Steps to reproduce the problem.
- What you expected to happen and what actually happened.
- Relevant logs, error messages, or a minimal reproducing example.

## Requesting Features

Feature requests are also welcome in the
[GitHub issue tracker](https://github.com/IntersectMBO/cardano-node-emulator/issues).
Please describe your use case and the problem you are trying to solve.

## Security Issues

Please **do not** report security vulnerabilities through public issues.
Instead, follow the process described in our [Security Policy](SECURITY.md).

## Contributing

Interested in contributing rather than just getting help? See
[CONTRIBUTING.md](CONTRIBUTING.md) and our [Governance](GOVERNANCE.md) document.
